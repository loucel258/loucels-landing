## 5b. Self-audit findings (post-session, before commit)

I re-read every route I modified looking for bugs typecheck can't catch. Findings:

- **Fixed live**: `/api/demo/hitl/decide` was not validating `id` as a UUID before
  passing it to Supabase. Added RFC-4122 regex check, returns 400 on malformed.
- **Watch (not changed)**: `decide()` in `lib/hitl/queue.ts` reads the proposal
  via service-role client without a `workspace_id` filter. The decide route does
  the cross-tenant guard BEFORE calling it, so the surface is safe today, but if
  `decide()` is ever called from a second path, that path MUST replicate the
  workspace check. Consider adding a required `workspace_id` arg to `decide()`
  before adding any new caller.
- **Watch (not changed)**: `/api/demo/auth` accepts arbitrary `role_label`
  strings without regex validation. The downstream RBAC route now validates
  against the `workspace_roles` table (P1 #7), so an invalid label fails fast
  there — but bad characters could pollute audit log `user_id` strings via the
  `agent_${role_label}_...` template. Add `/^[a-z][a-z0-9_]{2,40}$/` validation
  to the mint endpoint before opening it beyond ws_demo_001.
- **Watch (not changed)**: `isWorkspaceRoleValid` returns `true` when Supabase
  is not configured. That's the documented demo escape hatch but means a prod
  Supabase outage silently disables DB-side role checks. The TS enum check still
  applies, so it's not a security bypass — it's a feature degradation. Fine for
  now; revisit when adding a second workspace.

Everything else (request ordering, async/await, rate-limit before auth, JWT
re-check after long ops, audit-write fail-closed) looked correct.
